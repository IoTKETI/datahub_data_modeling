JSON-LD 예제 수정 이력 @190712
======================

1, 모든 엔티티 타입
- address: 도/시/동 등의 하위 속성을 별도로 두지 않고, value 속성에 JSON 객체로 정의
- observedAt (top-level) 속성 삭제

2. OffStreetParking
- contactPoint: telephone/email 등의 하위 속성을 별도로 두지 않고, value 속성에 JSON 객체로 정의
- locationTag, category, paymentAccepted, openingHours: values 하위 속성을 value로 변경
- availableSpotNumber: observedAt 하위 속성 추가
- refParkingSpots: Relation 타입에서 Property 타입으로 변경, objects 속성을 value 속성으로 변경
- congestionIndexPrediction: values 속성을 value로 변경, observedAt 하위 속성 추가

3. ParkingSpot
- refParkingLot: 타입을 Relationship에서 Property로 변경, object 속성을 value 속성으로 변경
- category: values 하위 속성을 value로 변경
- status: observedAt 하위 속성 추가

4. AirQualityObserved
- so2, co, o3, no2, pm10, pm25: 하나의 속성인 airQualityObservation으로 통합하고 하나의 JSON 객체로 정의, observedAt 하위 속성 추가
- airQualityIndex: 하위 속성들을 value 하나로 통합하고 JSON 객체로 변경, observedAt 하위 속성 추가
- location: observedAt 하위 속성 추가

5. AirQualityForecast
- airQualityIndexPrediction: values 속성을 value로 변경, indexRef 하위 속성 삭제, observedAt 속성 추가
- indexRef: top-level 속성으로 추가

6. WeatherObserved
- temperature, windSpeed, humidity, rainfall, hourlyRainfall, snowfall, visibility, weatherType: 하나의 속성인 weatherObservation으로 통합하고 하나의 JSON 객체로 정의, observedAt 하위 속성 추가, weather 값을 정수(Enum)에서 문자열로 수정
- location: observedAt 하위 속성 추가

7. WeatherForecast
- weatherPrediction: values 하위 속성을 value로 변경
